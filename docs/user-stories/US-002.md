## Story
As a client, I want to choose service type, gender, and location so that duration and price reflect my session before I confirm.

## Requirements
- Persist selections in the existing booking flow (`frontend/src/composables/useBookingSelection.ts` extended or companion state), carried into the confirm step (US-004).
- Service types: Regular haircut, Haircut + hairdressing, Color; gender: Male / Female; location: At salon / At home.
- Replace placeholder `frontend/src/views/client/ClientServiceView.vue` with a real form; **guard**: if no slot selected, redirect to `/`.
- **MVP pricing for this story:** Compute a **display** total the client sees: base from selected slot’s `price` (per US-001) plus a documented **constant** at-home surcharge (or `0` if already modeled — must match SPEC “adds an extra charge”). Wire to `PriceEntry` in US-012, not here.
  - Implementation: `AT_HOME_DISPLAY_SURCHARGE_EUR` in `frontend/src/constants/pricing.ts` (€12.50).
- **Duration:** Use `base_duration_minutes` from `Service` (spec); if the calendar slot window is shorter than required duration, either disable “Continue” with a clear message or filter in US-004 — pick one approach and document in acceptance criteria.
  - **Choice:** Disable “Continue” and show an inline warning on `ClientServiceView` when `slot.duration_minutes < service.base_duration_minutes`.
- **Backend:** Introduce `Service` (and enums) per `docs/SPEC.md` with seed data (pytest uses in-memory DB pattern from `backend/tests/conftest.py`).
- **Public read:** e.g. `GET /api/public/services` returning enough for the UI (type, gender, base duration, at-home buffer minutes for **internal** use later — may be omitted from JSON if buffer is kept server-only).

## Acceptance Criteria
- [x] `GET /api/public/services` returns 200 with non-empty list; enums/fields match SPEC.
- [x] Pytest: services exist after init/seed; response shape stable.
- [x] Vitest: `ClientServiceView` renders choices, disables continue until valid, navigates to `/book/confirm` (add route in US-002 or stub route name used by US-004).
- [x] Vitest: navigation `/book/service` without selected slot redirects home.
