## Story
As a client, I want to optionally upload a photo of my current hairstyle and an inspiration photo so that the hairdresser knows what I have and what I want.

## Requirements
- Upload step appears after service selection, before contact info
- Two optional upload fields: current hairstyle, inspiration
- Accepted formats: jpg, png, webp
- Files stored via the configured storage backend (local filesystem for dev)
- File ID stored on the booking record (current_hairstyle_media_id, inspiration_media_id)

## Acceptance Criteria
- [ ] `POST /api/public/media` accepts a file upload and returns a `media_id`
- [ ] API returns 400 if file format is not jpg, png, or webp
- [ ] `BookingMediaUpload.vue` renders two optional upload fields
- [ ] Submitting without uploading either image is valid (both are optional)
- [ ] Uploaded media_ids are passed along to the booking creation payload
- [ ] Test: `test_upload_valid_image_returns_media_id`
- [ ] Test: `test_upload_invalid_format_returns_400`
- [ ] Test: `BookingMediaUpload` renders correctly with no files selected
