## Story
As a client, I want to optionally leave a note for the hairdresser so that I can communicate any specific requests or context.

## Requirements
- Optional free text field on the booking form
- Appears on the same step as image upload (after service selection)
- Stored as client_note on the booking record
- Note is visible to the hairdresser in the back office

## Acceptance Criteria
- [ ] `POST /api/public/bookings` accepts an optional `client_note` field
- [ ] Note is stored on the booking record and returned in `GET /api/admin/bookings/{id}`
- [ ] Submitting without a note is valid
- [ ] Note is displayed in the back office booking detail view
- [ ] Test: `test_booking_with_note_stores_correctly`
- [ ] Test: `test_booking_without_note_is_valid`
- [ ] Test: `BookingNoteField` renders correctly with empty and filled states
