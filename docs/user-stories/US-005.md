## Story
As a client, I want to change my booking through my link so that I can reschedule or update details without calling the salon.

## Requirements
- **Public:** `GET /api/public/bookings/by-token/{token}`; `PUT` (or `PATCH`) with full replace per SPEC (“change everything”).
- While editing, **current slot is treated as available** for the new selection (recompute overlaps excluding this booking).
- On successful update: `status=edited` (per SPEC enum), re-bind slot, re-snapshot price/address if rules change, issue **new** `edit_token` **or** keep stable — **pick one** and document (stable token is simpler for emailed links).
- **Frontend:** Route e.g. `/book/edit/:token` multi-step reusing configure + confirm components or a dedicated wizard.

## Acceptance Criteria
- [x] Pytest: edit swaps slot and frees old slot; overlap rules still enforced.
- [x] Pytest: cannot edit cancelled booking.
- [x] Vitest: edit flow loads booking, submits update. (`ClientEditView.spec.ts`)

## Out of scope
- Rate limiting; token expiry (SPEC: no deadline).
