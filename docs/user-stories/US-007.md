## Context

## Story
As the hairdresser, I want to sign in securely so that only I can manage the salon data.

## Requirements
- Implement `backend/app/api/routers/admin_auth.py`: register (optional single-admin bootstrap), login, password hashing (e.g. passlib/bcrypt), issue signed token (JWT) or server session — **match** whatever `frontend/src/api/admin.ts` expects or update both together.
- Protect existing admin routers under dependency `backend/app/api/deps.py`.

## Acceptance Criteria
- [x] Pytest: login success/failure; protected route 401 without token.
- [x] Vitest: `frontend/src/views/admin/AdminLoginView.vue` submits credentials and stores token.

## Out of scope
- Multi-admin; password reset email.
