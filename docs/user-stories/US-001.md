## Story
As a client, I want to see available booking slots on a calendar so that I can pick a time that suits me.

## Requirements
- Display a weekly calendar view of available slots
- Each slot shows: date, start time, duration, and price
- Unavailable or already booked slots are visible but not selectable
- Clicking an available slot navigates to the service selection step (US-002)

## Acceptance Criteria
- [x] `GET /api/public/slots?week=YYYY-WW` returns a list of slots with `date`, `start_time`, `end_time`, `duration_minutes`, `price`, `is_available`
- [x] API returns 200 with an empty list when no slots exist for the requested week
- [x] `ClientHomeView.vue` renders one card per slot returned by the API
- [x] Slots where `is_available: false` are rendered as disabled (not clickable)
- [x] Clicking an available slot stores the selected slot in local state and navigates to `/book/service`
- [x] Test: `test_get_slots_returns_correct_week` — backend returns only slots matching the requested week
- [x] Test: `ClientHomeView` renders disabled state correctly when `is_available` is false

## Out of scope
- Slot creation (admin only, covered in a later story)
- Price calculation rules (flat price per slot for now, per SPEC.md MVP scope)
