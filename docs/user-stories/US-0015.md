## Story
As a hairdresser, I want to define my weekly working hours and mark exceptions so that the system can derive accurate availability for clients.

## Requirements
- Weekly availability is defined by a default open/close schedule per day of week (e.g. Mon 09:00-18:00)
- Exceptions override the default schedule for a specific datetime range (e.g. Tuesday afternoon closed)
- Exceptions have: start, end, created_at, comment (nullable)
- Available booking windows are derived from: weekly schedule minus exceptions minus existing bookings
- Service duration is taken into account when computing bookable slots (a 45 min service cannot fit in a 30 min window)
- Backend exposes a computed availability endpoint for clients

## Acceptance Criteria
- [ ] `WeeklySchedule` model exists with day_of_week, open_time, close_time
- [ ] `AvailabilityException` model exists with start, end, created_at, comment
- [ ] `GET /api/public/availability?service_id=X&week=YYYY-WW` returns bookable windows accounting for schedule, exceptions and existing bookings
- [ ] A window shorter than the selected service duration is excluded from results
