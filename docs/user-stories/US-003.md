## Story
As a developer, I want to populate SQLite with sample `Slot` rows for the current and next ISO weeks so that the client calendar is not empty when testing locally.

## Requirements
- Runnable from the repo with a documented command (from `backend/`, using the project virtualenv).
- Uses the same database URL and initialization path as the app (`app.config.settings.database_url` / `init_db` in `backend/app/db.py`) so the API and seed share one database.
- Inserts a small **deterministic** set of slots: mix of `is_available` true/false, varied start times and prices; covers **this ISO week** and **the next ISO week** so prev/next week navigation in the UI shows data.
- **Idempotency:** Before inserting, **delete all rows from the `slots` table**, then insert the demo set. (Dev-only; acceptable because this story is not production-facing.)

## Acceptance Criteria
- [x] Script entrypoint exists, e.g. `python -m app.seed_dev_slots` when run from `backend/` (exact module name as implemented).
- [x] After running the seed and starting the API, `GET /api/public/slots?week=<current ISO week>` returns a **non-empty** list whose items include the fields expected by the public slots contract (`date`, `start_time`, `end_time`, `duration_minutes`, `price`, `is_available`).
- [x] One-line “how to run” the seed is documented in the project `README.md` or `backend/README.md` (whichever the repo already uses for backend setup; add `backend/README.md` only if there is no suitable place).
- [x] **Test:** Pytest runs the seed routine against **in-memory SQLite** (same pattern as `backend/tests/conftest.py` / `StaticPool`), asserts a minimum row count after seeding, and asserts at least one slot falls within a given ISO week range aligned with `week=YYYY-WW` semantics.
